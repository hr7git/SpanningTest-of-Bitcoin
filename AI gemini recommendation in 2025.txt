## Chapter 3: Methodology

### 3.1 The Spanning Hypothesis

To determine if Bitcoin expands the investment opportunity set, we test if the **Efficient Frontier** of a portfolio of K traditional assets is the same as the frontier of K+1 assets (where the +1 is Bitcoin). If they are the same, we say the K assets "span" the K+1 assets, and Bitcoin adds no unique value.

### 3.2 Regression Model

We use a linear regression where the returns of Bitcoin (R_{btc,t}) are regressed against the returns of the benchmark assets (R_{1,t}, \dots, R_{k,t}):

Where:

* \alpha is the intercept (abnormal return).
* \beta_i represents the sensitivity to each traditional asset.
* \epsilon_t is the residual error.

### 3.3 The Two Constraints for Spanning

According to Huberman and Kandel, for Bitcoin to be "spanned" by the existing assets (meaning it adds **no** value), two conditions must be met simultaneously:

1. **\alpha = 0**: The new asset provides no extra return that isn't already explained by the others.
2. **\sum \beta_i = 1**: The new asset's risk/return profile can be perfectly replicated by a combination of the existing assets.

### 3.4 The Likelihood Ratio (LR) Test

We use the **F-test** or **Likelihood Ratio test** to evaluate the null hypothesis:


* **If we reject H_0:** It means Bitcoin **does** provide a unique risk-return profile that cannot be replicated by stocks, bonds, or gold. The investment boundary has expanded.
* **If we fail to reject H_0:** It means Bitcoin is redundant in a portfolio of these specific assets.

To perform the Huberman and Kandel (1987) spanning test, you need to run a **constrained multivariate regression**. In Python, the `statsmodels` library is excellent for the regression, and we can use a **Wald Test** to check the two constraints (\alpha = 0 and \sum \beta = 1) simultaneously.

Here is a Python script designed for this analysis.

### Python Script for Portfolio Spanning Test

```python
import pandas as pd
import numpy as np
import statsmodels.api as sm
from scipy import stats

# 1. Load your data
# Requirements: A CSV with columns 'Date', 'BTC', 'Stocks', 'Bonds', 'Gold', 'REITs'
# Returns should be in percentage change (decimal or percentage)
df = pd.read_csv('portfolio_data.csv', index_index='Date', parse_dates=True)

# Define variables
# Bitcoin is the dependent variable (y)
y = df['BTC']
# Traditional assets are the independent variables (X)
X = df[['Stocks', 'Bonds', 'Gold', 'REITs']]

# Add a constant to X for the Alpha (intercept) calculation
X_with_const = sm.add_constant(X)

# 2. Fit the OLS Regression
model = sm.OLS(y, X_with_const).fit()

print("--- Regression Results ---")
print(model.summary())

# 3. Perform the Huberman and Kandel Spanning Test (Wald Test)
# We test two hypotheses:
# 1) The constant (const) is 0
# 2) The sum of all asset coefficients (beta_i) is 1
hypotheses = '(const = 0), (Stocks + Bonds + Gold + REITs = 1)'
wald_test = model.wald_test(hypotheses)

print("\n--- Wald Test for Spanning ---")
print(f"F-statistic: {wald_test.fvalue[0][0]:.4f}")
print(f"p-value: {wald_test.pvalue:.4f}")

# 4. Interpretation
if wald_test.pvalue < 0.05:
    print("\nResult: Reject the Null Hypothesis (p < 0.05)")
    print("Interpretation: Bitcoin provides significant expansion to the portfolio frontier.")
else:
    print("\nResult: Fail to Reject the Null Hypothesis")
    print("Interpretation: Bitcoin is spanned by existing assets; no significant expansion.")

```

### How to Interpret the Output

When you run this script, pay close attention to the **p-value** of the Wald Test:

1. **If p-value < 0.05:** You have statistical evidence that the **Minimum-Variance Frontier** has shifted. This means Bitcoin provides a risk-return profile that cannot be replicated by just holding a mix of stocks, bonds, and gold.
2. **The Alpha (\alpha):** If the alpha is positive and significant in the regression summary, Bitcoin is adding "excess return" beyond what is explained by market movements.
3. **The Betas (\beta):** These will tell you which assets Bitcoin is most correlated with. A high beta for gold would suggest it behaves like a commodity; a high beta for stocks suggests it is a "risk-on" asset.
